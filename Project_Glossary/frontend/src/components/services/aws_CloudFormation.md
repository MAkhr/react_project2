# AWS CloudFormation

インフラストラクチャーをコード化し、AWSリソースのプロビジョニングと管理を自動化するサービスです。
テキストファイル（テンプレート）にリソースの設定を記述し、CloudFormationがそれに基づいてリソースの作成、更新、削除を行う。

## 主な特徴
- テンプレートは、JSONまたはYAML形式で記述
- テンプレート内のリソース間の依存関係を自動的に解決し、リソースを正しい順序で作成
- スタックを作成、更新、削除することで、インフラストラクチャーの変更が管理される
- CloudFormationは自動的にロールバックを行い、以前の成功した状態に戻す
- Change Setsはスタックの変更内容をプレビューする機能
- CloudFormationはAWSのすべてのリソースがサポートされているわけではない。
- AWS CloudFormationでは、サポートされていないリソースや独自のリソースをカスタムリソースとして定義し、AWS Lambda関数と組み合わせて利用することができる
